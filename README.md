# 数据记录

这是一个手机优先的小型 PWA 仪表盘，用来记录每日数值和阶段进度。

## 每天怎么用

在 Codex 里发：

```text
今天数值 102xxxx.xx
```

Codex 更新 `data.json` 后，刷新页面即可看到新的数值、日变化、阶段进度和线形图。

也可以直接在页面里的“今日记录”录入数值；这会保存在当前浏览器本地。

首页的走势图支持“阶段 / 变化”和“日 / 周 / 月 / 年”切换。手指在图上滑动或点按数据点时，下方会显示对应日期或周期的数值、变化、累计变化和离阶段差额。

本地更新数据也可以用：

```bash
node update-value.mjs 2026-07-08 1030000
```

## 手机查看

在同一 Wi-Fi 下，可用 Mac 的局域网地址访问本地预览。要长期像 App 一样使用，建议把这个文件夹托管成一个固定 HTTPS 链接，然后在 iPhone Safari 里添加到主屏幕。

## GitHub Pages 发布

这个文件夹已经整理成 GitHub Pages 可直接发布的静态站点。推荐创建一个不容易猜到名字的公开仓库，例如 `daily-data-log-2026`，把本文件夹里的所有文件上传到仓库根目录，然后在仓库 `Settings -> Pages` 里选择 `Deploy from a branch`，分支选 `main`，目录选 `/root`。

发布后地址通常是：

```text
https://你的GitHub用户名.github.io/仓库名/
```

在 iPhone Safari 打开这个地址，点分享按钮，再点“添加到主屏幕”，就能像轻 App 一样打开。

隐私提醒：GitHub Free 的 Pages 通常需要公开仓库。这个版本加了 `robots.txt` 和 `noindex`，可以降低被搜索引擎收录的概率，但不能当作真正的隐私保护；知道链接或打开仓库的人仍然可能看到 `data.json` 里的数值数据。

## 数据源

- 默认数据：`data.json`
- 本地临时输入：浏览器 localStorage
- 阶段阶梯：`data.json` 里的 `targetSteps`

阶段按“较基准数值的累计变化”计算。
首页展示“还差数值”，阶段页里的“阶段门槛”表示“基准数值 + 当前阶段变化”。
